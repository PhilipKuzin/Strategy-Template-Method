# Strategy-Template-Method

В данном репозитории представлено 6 различных вариантов применения паттернов "Стратегия" и "Шаблонный метод" на примере различных игровых механик. 
1) На сцене IW1 реализована подмена поведения (патруль, преследование, остановка) объекта "Шериф" в зависимости от реакции на пользовательский ввод.
2) На сцене IW2 показан пример совместного использования паттернов "Стратегия" и "Шаблонный метод": наследники абстрактного класса Character переопределяют метод Affect, получая различные варианты воздействия друг на друга (удар топором/кулаком). Класс CheckerExample определяет варианты инициализации персонажей предоставляя возможность подменять логику поиска противников персонажами (FrontView, CircleView, NoView). 
3) На сцене IW3 представлен вариант применения паттерна "Шаблонный метод" на примере мини-игры "CoinPicker". Наследники абстрактного класса Coin переопределяют метод Add, добавляя персонажу различное количество очков в зависимости от типа подобранной монеты.
4) На сцене HW2 представлено 3 типа оружия (наследники абстрактного класса WeaponAbstract), переопределяющие метод Shoot, и игрок, имеющий возможность поднимать, бросать оружие и производить выстрел. 
5) На сцене HW3 реализована подмена поведения торговца (не торгует/торгует фруктами/торгует броней) в зависимости от репутации игрока, настраиваемой пользователем через элементы UI. 
6) На сцене HW4 продемонстрирована мини-игра "Pop The Balloon". До старта игры пользователь выбирает условия выигрыша (подмена условия происходит на основе паттерна "Стратегия"): лопнуть все шары/лопнуть шары только определенного цвета.
